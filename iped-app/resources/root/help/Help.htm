Search Syntax:

To open the analysis application, it is required to install Java JRE 8 or newer, a 64bits JRE is highly recommended, download from www.java.com. If your case or report is stored in an optical disk, analysis can become slow. If so, copy the whole case to a local hard disk, from where the analysis application should be opened.

The keyword search looks for terms in file name and contents of textual files, like Word, Excel, PDF, Emails, Internet pages, chats, so on. If enabled, OCR (Optical Character Recognition) technology is used to search for text in image files. However, some results may not be found in low quality or low resolution images, or if fonts are small or uncommon.

The keyword search does not differentiate letter capitalization or accents. But it is recommended to include accents in searches, so keywords will be properly highlighted in "Preview" tab. It is not recommended to search for common or very short keywords, like OF. Below are some search examples:
  • 1. suspect
    Search for files containing the exact term suspect. Files containing suspects will not be found.

  • 2. suspect?
    Search for files containing words starting with suspect plus any letter at the end, like suspects.

  • 3. suspect*
    Search for files containing words starting with suspect with ou without a suffix, like suspect, suspects and suspecting.

  • 4. Sicrano~
    Search for files containing words equal or similar to sicrano, like siclano, cicrano and ciclano. It is slower, but its use is recommended to search for text in images (if OCR was enabled).

  • 5. "Thomas Edson"
    Search for the exact expression Thomas Edson. It is not possible to use * ? ~ between double quotes.

  • 6. Thomas Edson
    Search for files containing any of the words Thomas or Edson. It is equal to searching for Thomas OR Edson

  • 7. Thomas AND Edson
    Search for both Thomas and Edson

  • 8. bribery -payment
    Search for files containing bribery but with no references to payment. It is equal to searching for bribery AND NOT payment

  • 9. (Thomas Edson) && payment
    Search for files containing payment AND any of Thomas OR Edson.

  • 10. "Thomas Edson"~5
    Search for files containing Thomas at most 5 words of distance of Edson.

  • 11. name:receipt
    Search for files containing receipt in its name property.
    All properties shown in the columns manager can be used, except for "bookmark" e "score" currently.

  • 12. size:[5000 TO *]
    Search for files with size greater than 5000 bytes, inclusive.

  • 13. modified:[2015-01-31 TO 2015-03-31T12\:59\:59Z]
    Search for files with modification date between 12:00am of 01/31/2015 and 12:59:59pm of 03/31/2015.

  • 14. /<0-17>y(r|o)/
    Search for regex, delimited by 2 forward slashes. It only works for indexed letters (alphanumeric by default).

For more information about search syntax, see http://lucene.apache.org/core/4_9_0/queryparser/org/apache/lucene/queryparser/classic/package-summary.html.
