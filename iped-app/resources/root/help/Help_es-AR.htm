Sintaxis de búsqueda:

Para abrir la interfaz de análisis, necesita un sistema operativo de 64 bits. Si su caso o informe está almacenado en un disco óptico, el análisis puede volverse lento. Si es así, copie todo el caso en un disco duro local, desde donde se debe abrir la aplicación de análisis..

La búsqueda por palabra clave busca términos en el nombre del archivo y el contenido de los archivos de texto, como Word, Excel, PDF, correos electrónicos, páginas de Internet, chats, etc. Si está habilitado, OCR (Reconocimiento óptico de caracteres) La tecnología se utiliza para buscar texto en archivos de imagen. Sin embargo, es posible que algunos resultados no se encuentren en imágenes de baja calidad o baja resolución, o si las fuentes son pequeñas o poco comunes.

La búsqueda por palabra clave no diferencia entre mayúsculas y minúsculas o acentos. Pero se recomienda incluir acentos en las búsquedas, de modo que las palabras clave se resalten correctamente en la pestaña "Vista previa".. No se recomienda buscar palabras clave comunes o muy cortas, como OF. A continuación se muestran algunos ejemplos de búsqueda:
  • 1. sospecha
    Buscar archivos que contengan el término exacto sospecha. Archivos que contienen sospechoso no se encontrarán.

  • 2. sospecha?
    Buscar archivos que contengan palabras que empiecen por sospecha más cualquier letra al final, como sospechoso.

  • 3. sospecha*
    Buscar archivos que contengan palabras que empiecen por sospecha con ou sin sufijo, como sospecha, sospechoso y sospechado.

  • 4. Sicrano~
    Busca archivos que contengan palabras iguales o similares a sicrano, como sicrano, cicrano y < b>ciclano. Es más lento, pero se recomienda su uso para buscar texto en imágenes (si se habilitó OCR).

  • 5. "Thomas Edson"
    Busque la expresión exacta Thomas Edson. No es posible usar * ? ~ entre comillas dobles.

  • 6. Thomas Edson
    Busca archivos que contengan cualquiera de las palabras Thomas o Edson. Es igual a buscar Thomas OR Edson

  • 7. Thomas AND Edson
    Busca tanto Thomas como Edson

  • 8. soborno -pago
    Busca archivos que contengan soborno pero sin referencias a pago. Es igual a buscar soborno Y NO pago

  • 9. (Thomas Edson) && pago
    Busca archivos que contengan pago Y cualquiera de Thomas O Edson.

  • 10. "Thomas Edson"~5
    Busca archivos que contengan Thomas como máximo 5 palabras de distancia de Edson.

  • 11. (Thomas Edson John)@2
    Busque archivos que contengan al menos dos términos del conjunto.

  • 12. nombre:recibo
    Busca archivos que contengan recibo en su propiedad de nombre.
    Todas las propiedades que se muestran en el administrador de columnas se pueden utilizar, excepto "marcador" y "puntuación" actualmente.

  • 13. tamaño:[5000 TO *]
    Busca archivos con un tamaño superior a 5000 bytes, inclusive. Utilice {} en lugar de [] para excluir el valor.

  • 14. modificado:[2015-01-31 TO 2015-03-31T12\:59\:59Z]
    Buscar archivos con fecha de modificación entre las 00:00 horas del 31/01/2015 y el 12: 59:59pm del 31/03/2015.

  • 15. /<0-17>y(r|o)/
    Busca expresiones regulares, delimitadas por 2 barras diagonales. Solo funciona para letras indexadas (alfanuméricas por defecto).

Para obtener más información sobre la sintaxis de búsqueda, consulte https://lucene.apache.org/core/9_2_0/queryparser/org/apache/lucene/queryparser/flexible/standard/StandardQueryParser.html.
