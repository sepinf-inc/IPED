Guida rapida IPED

Sintassi di ricerca:

Per aprire l'interfaccia di analisi, è necessario un sistema operativo a 64 bit. Se il vostro Caso o Report è memorizzato in un disco ottico, l'analisi può diventare lenta. In tal caso, copiate l'intero Caso su un disco rigido locale, da dove aprire l'applicazione di analisi.

La ricerca per parole chiave cerca i termini nel nome del file e nel contenuto dei file testuali, come Word, Excel, PDF, e-mail, pagine Internet, chat, ecc.. Se abilitata, la tecnologia OCR (Optical Character Recognition) viene utilizzata per cercare il testo nei file di immagine. Tuttavia, alcuni risultati potrebbero non essere trovati in immagini di bassa qualità o bassa risoluzione, o se i caratteri sono piccoli o poco comuni.

La ricerca per parole chiave non distingue la capitalizzazione delle lettere o gli accenti. Si raccomanda, tuttavia, di includere gli accenti nelle ricerche, così le parole chiave vengano evidenziate correttamente nella scheda "Anteprima". Non è raccomandato cercare parole chiave comuni o molto brevi, come DI. Di seguito sono riportati alcuni esempi di ricerca:
  • 1. sospetto
    Cerca i file che contengono la parola esatta sospetto. I file contenenti la parola sospetti non verranno trovati.

  • 2. sospett?
    Cerca i file che contengono parole che iniziano con sospett più una qualsiasi lettera alla fine, come sospetti o sospetta ecc..

  • 3. sospett*
    Cerca i file contenenti parole che iniziano con sospett con o senza un suffisso, come sospetto, sospetti e sospettoso ecc..

  • 4. carta~
    Cerca file contenenti parole uguali o simili a carta, come casta, canta e salta. La ricerca è più lenta, ma il suo uso è raccomandato per trovare il testo nelle immagini o documenti scansionati (se l'OCR è stato abilitato).

  • 5. "Mario Rossi"
    Cerca l'espressione esatta Mario Rossi. Non è possibile usare i caratteri jolly * ? ~ tra doppi apici.

  • 6. Mario Rossi
    Cerca i file che contengono una qualsiasi delle parole Mario o Rossi. È uguale alla ricerca di Mario OR Rossi

  • 7. Mario AND Rossi
    Cerca sia Mario che Rossi

  • 8. corruzione -pagamento
    Cerca i file contenenti corruzione ma senza riferimenti a pagamento. È uguale alla ricerca di corruzione AND NOT pagamento

  • 9. (Mario Rossi) && pagamento
    Cerca i file che contengono sia la parola pagamento sia una qualsiasi tra le parole Mario e Rossi.

  • 10. "Mario Rossi"~5
    Cerca i file che contengono Mario al massimo a 5 parole di distanza da Rossi.

  • 11. (Mario Luigi Rossi)@2
    Cerca i file contenenti almeno due termini del set.

  • 12. name:ricevuta
    Cerca i file che contengono ricevuta nella proprietà name.
    Tutte le proprietà mostrate nel gestore delle colonne possono essere usate, tranne "bookmark" e "score" per adesso.

  • 13. size:[5000 TO *]
    Cerca i file con dimensioni maggiori o uguali di 5000 bytes. Utilizzare {} invece di [] per escludere il valore.

  • 14. modified:[2015-01-31 TO 2015-03-31T12\:59\:59Z]
    Cerca i file con data ultima modifica compresa tra le 12:00am del 31/01/2015 e le 12:59:59pm del 31/03/2015.

  • 15. /<0-17>y(r|o)/
    Cerca un'espressione RegEx, delimitata da 2 barre in avanti "/.../". Funziona solo per i caratteri indicizzati (alfanumerici per impostazione predefinita).

Per ulteriori informazioni sulla sintassi di ricerca, visita https://lucene.apache.org/core/9_2_0/queryparser/org/apache/lucene/queryparser/flexible/standard/StandardQueryParser.html.
