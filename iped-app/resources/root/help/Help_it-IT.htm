IPED Short Help

Sintassi di ricerca:

Per aprire l'applicazione di analisi, è necessario installare Java JRE 8 o più recente, JRE a 64 bit è altamente raccomandato, scaricabile da www.java.com. Se il vostro Caso o Report è memorizzato in un disco ottico, l'analisi può diventare lenta. In tal caso, copiate l'intero Caso su un disco rigido locale, da dove aprire l'applicazione di analisi.

La ricerca per parole chiave cerca i termini nel nome del file e nel contenuto dei file testuali, come Word, Excel, PDF, e-mail, pagine Internet, chat, ecc.. Se abilitata, la tecnologia OCR (Optical Character Recognition) viene utilizzata per cercare il testo nei file di immagine. Tuttavia, alcuni risultati potrebbero non essere trovati in immagini di bassa qualità o bassa risoluzione, o se i caratteri sono piccoli o poco comuni.

La ricerca per parole chiave non distingue la capitalizzazione delle lettere o gli accenti. Si raccomanda, tuttavia, di includere gli accenti nelle ricerche, così le parole chiave vengano evidenziate correttamente nella scheda "Anteprima". Non è raccomandato cercare parole chiave comuni o molto brevi, come DI. Di seguito sono riportati alcuni esempi di ricerca:
  • 1. sospetto
    Cerca i file che contengono il termine esatto sospetto. I file contenenti sospetti non verranno trovati.

  • 2. sospett?
    Cerca i file che contengono parole che iniziano con sospett più una qualsiasi lettera alla fine, come sospetti.

  • 3. sospett*
    Cerca i file contenenti parole che iniziano con sospetto con o senza un suffisso, come sospetto, sospetti e sospettoso.

  • 4. carta~
    Cerca file contenenti parole uguali o simili a carta, come casta, canta e salta. È più lento, ma il suo uso è raccomandato per cercare il testo nelle immagini (se l'OCR è stato abilitato).

  • 5. "Mario Rossi"
    Cerca l'espressione esatta Mario Rossi. Non è possibile usare * ? ~ tra doppi apici.

  • 6. Mario Rossi
    Cerca i file che contengono una qualsiasi delle parole Mario o Rossi. È uguale alla ricerca di Mario OR Rossi

  • 7. Mario AND Rossi
    Cerca sia Mario che Rossi

  • 8. corruzione -pagamento
    Cerca i file contenenti corruzione ma senza riferimenti a pagamento. È uguale alla ricerca di corruzione AND NOT pagamento

  • 9. (Mario Rossi) && pagamento
    Cerca i file che contengono pagamento AND uno qualsiasi tra Mario OR Rossi.

  • 10. "Mario Rossi"~5
    Cerca i file che contengono Mario al massimo a 5 parole di distanza da Rossi.

  • 11. name:ricevuta
    Cerca i file che contengono ricevuta nella proprietà name.
    Tutte le proprietà mostrate nel gestore delle colonne possono essere usate, tranne "bookmark" e "score" per adesso.

  • 12. size:[5000 TO *]
    Cerca i file con dimensioni maggiori o uguali di 5000 bytes.

  • 13. modified:[2015-01-31 TO 2015-03-31T12\:59\:59Z]
    Cerca i file con data di modifica compresa tra le 12:00am del 31/01/2015 e le 12:59:59pm del 31/03/2015.

  • 14. /<0-17>y(r|o)/
    Cerca un'espressione RegEx, delimitata da 2 barre in avanti "/.../". Funziona solo per i caratteri indicizzati (alfanumerici per impostazione predefinita).

Per ulteriori informazioni sulla sintassi di ricerca, visita http://lucene.apache.org/core/4_9_0/queryparser/org/apache/lucene/queryparser/classic/package-summary.html.
