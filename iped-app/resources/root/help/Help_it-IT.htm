Sintassi di ricerca:

Per aprire l'interfaccia di analisi, è necessario un sistema operativo a 64 bit. Se il vostro Caso o Report è memorizzato in un disco ottico, l'analisi potrebbe diventare lenta. In tal caso, copiate l'intero Caso su un disco rigido locale, da cui aprire l'applicazione di analisi.

La ricerca per parole chiave cerca i termini nel nome del file e nel contenuto dei file testuali, come Word, Excel, PDF, e-mail, pagine Internet, chat, ecc.. Se abilitata, viene utilizzata la tecnologia OCR (Riconoscimento Ottico dei Caratteri) per cercare il testo nei file immagine. Tuttavia, alcuni risultati potrebbero non essere trovati in immagini di bassa qualità o bassa risoluzione, o se i caratteri sono piccoli o poco comuni.

La ricerca per parole chiave non distingue la capitalizzazione delle lettere o gli accenti. Tuttavia, si consiglia di includere gli accenti nelle ricerche, in modo che le parole vengano evidenziate correttamente nella scheda "Anteprima". Non è consigliato cercare parole chiave comuni o molto brevi, come "DI". Di seguito sono riportati alcuni esempi di ricerca:
  • 1. sospetto
    Cerca i file contenenti la parola esatta sospetto. I file contenenti la parola sospetti o sospettoso non verranno trovati.

  • 2. sospett?
    Cerca i file contenenti parole che iniziano con sospett più una qualsiasi lettera alla fine, come sospetti o sospetta ecc..

  • 3. sospett*
    Cerca i file contenenti parole che iniziano con sospett con o senza un suffisso, come sospetto, sospetti e sospettoso ecc..

  • 4. carta~
    Cerca file contenenti parole uguali o simili a carta, come casta, canta e salta. La ricerca è più lenta, ma il suo utilizzo è consigliato per trovare il testo nelle immagini o documenti scansionati (se l'OCR è stato abilitato).

  • 5. "Mario Rossi"
    Cerca l'espressione esatta Mario Rossi. Non è possibile usare i caratteri jolly * ? ~ tra virgolette.

  • 6. Mario Rossi
    Cerca i file contenenti una delle parole Mario o Rossi. Equivale a cercare Mario OR Rossi

  • 7. Mario AND Rossi
    Cerca file contenenti sia Mario che Rossi

  • 8. corruzione -pagamento
    Cerca i file contenenti corruzione ma escludendo quelli contenenti pagamento. Equivalente a cercare corruzione AND NOT pagamento

  • 9. (Mario Rossi) && pagamento
    Cerca i file contenenti sia la parola pagamento sia una delle parole Mario o Rossi.

  • 10. "Mario Rossi"~5
    Cerca i file contenenti Mario a massimo 5 parole di distanza da Rossi.

  • 11. (Mario Luigi Rossi)@2
    Cerca i file contenenti almeno due termini del set.

  • 12. nome:ricevuta
    Cerca i file contenenti ricevuta nella proprietà Nome.
    Tutte le proprietà mostrate nel gestore delle colonne possono essere usate, tranne "Segnalibri" e "Percentuale" per adesso.

  • 13. dimensione:[5000 TO *]
    Cerca i file con dimensione maggiore o uguale a 5000 bytes. Utilizzare {} invece di [] per escludere il valore.

  • 14. modificato:[2015-01-31 TO 2015-03-31T12\:59\:59Z]
    Cerca file con data di modifica compresa tra le 12:00am del 31/01/2015 e le 12:59:59pm del 31/03/2015.

  • 15. /<0-17>y(r|o)/
    Cerca un'espressione RegEx, delimitata da 2 barre in avanti "/.../". Funziona solo per i caratteri indicizzati (alfanumerici per impostazione predefinita).

Per ulteriori informazioni sulla sintassi di ricerca, visita https://lucene.apache.org/core/9_2_0/queryparser/org/apache/lucene/queryparser/flexible/standard/StandardQueryParser.html.
